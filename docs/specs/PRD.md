# 产品需求文档（PRD）

> Product Requirements Document — EasyTransfer
> 版本：0.1.0 | 最后更新：2026-03-25

---

## 1. 产品背景

### 1.1 问题描述

当用户购买新的 Windows 电脑后，面临一个极其痛苦的过程：将旧电脑上积累的所有软件、配置、文件和凭证迁移到新电脑上。目前这个过程存在以下痛点：

- **耗时巨大**：通常需要 1-3 天才能让新电脑达到旧电脑的工作状态
- **容易遗漏**：总会忘记某些配置或某个小工具，直到用的时候才发现
- **技术门槛高**：很多配置需要专业知识才能正确迁移
- **没有好用的现有工具**：Windows 易传输已被移除，第三方工具要么贵、要么功能不完整

### 1.2 目标用户

**Phase 1 核心用户**：程序员 / 开发者
- 电脑上有大量开发工具和复杂配置
- 对迁移完整性要求最高
- 有能力运行命令行工具
- 愿意为节省时间付费

**Phase 2 扩展用户**：普通知识工作者
- 办公软件 + 浏览器为主
- 需要图形界面
- 对操作简单性要求更高

### 1.3 产品愿景

> 像手机换机一样简单地完成电脑换机 —— 在新电脑上一键恢复旧电脑的一切。

## 2. 产品功能

### 2.1 核心功能（Must Have）

#### F1：智能环境扫描
- **描述**：自动扫描旧电脑上的所有已安装应用、用户文件、系统配置和凭证信息
- **智能点**：AI Agent 能理解哪些内容值得迁移，自动过滤临时文件、缓存等垃圾
- **输出**：一份结构化的"电脑环境画像"

**扫描范围包括：**

| 类别 | 具体内容 | 优先级 |
|------|---------|--------|
| 已安装应用 | 应用名称、版本、安装路径、安装来源 | P0 |
| 应用配置 | 各应用的用户配置文件（settings、preferences 等） | P0 |
| 用户文件 | 文档、下载、桌面、图片等用户目录下的文件 | P0 |
| 浏览器数据 | 书签、扩展列表、已保存的密码（加密） | P0 |
| 开发环境 | Python/Node/Java 等运行时、包管理器、全局包列表 | P0 |
| Git 配置 | .gitconfig、SSH 密钥、GPG 密钥 | P0 |
| 系统配置 | 环境变量、hosts 文件、系统主题、壁纸 | P1 |
| 网络配置 | WiFi 密码、VPN 配置、代理设置 | P1 |
| 终端配置 | Windows Terminal 配置、PowerShell profile | P1 |
| 计划任务 | Windows 计划任务 | P2 |
| 字体 | 用户安装的非系统字体 | P2 |

#### F2：迁移计划生成
- **描述**：AI Agent 分析扫描结果，生成一份详细的迁移计划
- **智能点**：Agent 能判断每个应用的最佳安装方式，识别可能的兼容性问题
- **用户交互**：用户可以审核计划，勾选/取消特定项目，修改迁移策略
- **输出**：一份用户确认过的迁移执行计划

**迁移计划示例：**

```
=== EasyTransfer 迁移计划 ===

📦 应用安装（23 个应用）
  ✅ 自动安装（18 个）：
     - Google Chrome → winget install Google.Chrome
     - Visual Studio Code → winget install Microsoft.VisualStudioCode
     - Python 3.11 → winget install Python.Python.3.11
     - ...

  ⚠️ 需要手动操作（3 个）：
     - Adobe Photoshop → 需要登录 Adobe 账号重新安装
     - 企业VPN客户端 → 需要联系 IT 部门获取安装包
     - 某个古老的小工具 → 已停止更新，建议替代方案：xxx

  ❌ 建议不迁移（2 个）：
     - 旧版驱动管理器 → 新电脑不需要
     - 临时安装的测试软件 → 已 6 个月未使用

📁 文件传输（总计 45.2 GB）
  - 文档：12.3 GB
  - 代码项目：8.7 GB（跳过 node_modules 等依赖目录，节省 15 GB）
  - 图片：5.1 GB
  - ...

⚙️ 配置恢复（35 项）
  - VS Code 设置 + 30 个扩展
  - Git 全局配置 + SSH 密钥
  - Windows Terminal 配置
  - ...

🔐 凭证迁移（需要您确认）
  - SSH 密钥（3 个）
  - Chrome 已保存密码（建议使用 Google 账号同步）

⏱️ 预计耗时：约 45 分钟（取决于网络速度）
```

#### F3：安全数据传输
- **描述**：在旧电脑和新电脑之间建立安全连接，传输迁移数据
- **传输方式**：优先局域网 P2P 直连（速度最快），备选方案为中继服务器
- **安全要求**：TLS 加密传输，凭证类数据额外一层 AES-256 加密
- **功能**：支持断点续传、传输进度显示、带宽限制（不影响正常上网）

#### F4：智能环境重建
- **描述**：在新电脑上按照迁移计划自动安装应用、恢复配置、传输文件
- **智能点**：遇到问题时 Agent 能自动尝试修复，无法修复时给出清晰的说明
- **执行方式**：逐步执行，每完成一个大步骤给用户反馈
- **验证**：每个步骤完成后自动验证（如：应用能否正常启动、配置文件是否生效）

#### F5：迁移报告
- **描述**：迁移完成后生成一份详细报告
- **内容**：成功项、失败项（附原因和手动修复指引）、跳过项、统计信息
- **格式**：HTML 格式，用户可以在浏览器中查看

### 2.2 增强功能（Should Have）

#### F6：迁移回滚
- 在新电脑上可以撤销全部或部分迁移操作
- 每个步骤都记录回滚信息

#### F7：选择性迁移
- 用户可以只选择迁移特定类别（如只迁移开发环境）
- 支持"配置模板"（如"开发者模式"、"办公模式"）

#### F8：迁移进度保存
- 支持中断后恢复迁移（如电脑意外关机）
- 迁移状态持久化到磁盘

### 2.3 未来功能（Nice to Have）

#### F9：跨系统迁移（Windows → Mac）
#### F10：图形化界面（GUI）
#### F11：云端备份迁移画像（不经过两台电脑直连）

## 3. 用户流程

### 3.1 主流程

```
用户视角的完整流程：

第 1 步：在旧电脑和新电脑上分别安装 EasyTransfer
        （一个安装包，运行后选择"我是旧电脑"或"我是新电脑"）

第 2 步：两台电脑连接同一网络，EasyTransfer 自动发现对方
        （显示配对码让用户确认，防止连错电脑）

第 3 步：旧电脑开始智能扫描（约 2-5 分钟）
        （显示扫描进度：正在扫描应用... 正在扫描配置... 正在分析文件...）

第 4 步：显示迁移计划，用户审核并确认
        （用户可以勾选/取消特定项，修改文件筛选条件等）

第 5 步：开始迁移！
        （显示整体进度和当前正在进行的操作）
        （重要操作会弹出确认框，如"是否迁移SSH密钥？"）

第 6 步：迁移完成，显示迁移报告
        （成功了多少，失败了多少，需要手动处理的有哪些）

第 7 步：用户根据报告中的指引处理少量需要手动操作的项目
```

### 3.2 异常流程

| 异常情况 | 处理方式 |
|---------|---------|
| 网络断开 | 自动暂停，网络恢复后从断点继续 |
| 旧电脑关机 | 新电脑保存已接收数据，旧电脑重启后继续 |
| 新电脑磁盘空间不足 | 提前检测并警告，让用户选择跳过部分文件 |
| 应用安装失败 | 记录失败原因，继续其他步骤，最终在报告中列出 |
| 用户取消迁移 | 提供选项：保留已迁移内容 / 回滚到迁移前状态 |

## 4. 非功能需求

### 4.1 性能要求
- 扫描阶段：5 分钟内完成（普通电脑，约 100 个应用 + 100GB 文件）
- 传输速度：局域网下应达到网络带宽的 80% 以上
- 内存占用：不超过 500MB

### 4.2 安全要求
- 所有网络传输必须 TLS 1.3 加密
- 密码和密钥类数据必须额外 AES-256 加密
- 配对过程需要用户在两端确认（防止中间人攻击）
- 迁移完成后，传输通道中的临时数据必须清除
- 不收集、不上传任何用户数据到我们的服务器

### 4.3 兼容性要求
- 支持 Windows 10（1903+）和 Windows 11
- 支持 x64 和 ARM64 架构
- 旧电脑和新电脑可以是不同的 Windows 版本

### 4.4 可靠性要求
- 文件传输必须校验哈希值，确保数据完整性
- 迁移过程支持断点续传
- 单个步骤失败不影响其他步骤执行

## 5. 成功指标

| 指标 | 目标值 |
|------|--------|
| 应用识别率 | ≥ 95%（能识别出旧电脑上 95% 的已安装应用） |
| 自动安装成功率 | ≥ 80%（识别出的应用中，80% 能自动安装成功） |
| 配置恢复准确率 | ≥ 90%（迁移的配置在新电脑上能正确工作） |
| 文件传输完整性 | 100%（零数据丢失） |
| 端到端迁移时间 | ≤ 2 小时（100GB 数据 + 50 个应用，千兆局域网） |
| 用户满意度 | 迁移后无需手动处理的项目占比 ≥ 70% |

## 6. 术语表

| 术语 | 定义 |
|------|------|
| 源端 / Source | 旧电脑，数据的来源 |
| 目标端 / Target | 新电脑，数据的目的地 |
| 环境画像 / Profile | 对一台电脑软硬件环境的结构化描述 |
| 迁移计划 / Migration Plan | AI Agent 生成的、经用户确认的迁移操作清单 |
| 迁移项 / Migration Item | 迁移计划中的一个独立操作单元（如"安装 Chrome"） |
| Agent | 运行在电脑上的 AI 智能代理程序 |
